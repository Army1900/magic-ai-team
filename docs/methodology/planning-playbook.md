# Planning Playbook

## Discovery Questions
- What exact problem should this team solve?
- What outcome defines success?
- What constraints exist (cost, latency, compliance, tools)?
- How much human approval is required?

## Team Design Rules
- Start with minimum viable roles.
- Separate planning, execution, and evaluation responsibilities.
- Map skills and MCPs only when needed for target tasks.

## Model Selection Rules
- Quality-first tasks use stronger models.
- High-volume tasks prioritize lower-cost models.
- Always define fallback models.
